## Purpose

Reqqy is a web-based Laravel application that enables users to capture feature requests and new application ideas through a conversational AI business analyst, automatically generating professional Product Requirement Documents (PRDs) for review and implementation. The primary users are end users requesting features and admins who review and approve conversations and PRDs. It solves the problem of extracting clear, complete requirements from stakeholders by replacing forms and emails with guided, natural-language conversations that produce structured documentation.

## Tech Stack

Laravel 12.0, PHP 8.4, livewire, flux, flux-pro

## Features (from tests)

- Generates a PRD document from conversation messages.
- Creates a document even if no messages exist.
- Notifies all admin users when a document is created.
- Includes conversation link in document notifications.
- Displays conversation summary and signed-off timestamp.
- Displays all conversation messages and message counts.
- Can create a new conversation and load existing conversations (with or without application_id).
- Prevents sending messages after a conversation is signed off.
- Can sign off a conversation and prevents signing off again.
- Dispatches relevant PRD and research jobs on conversation sign-off (e.g., generate new application PRD, generate feature request PRD, research alternatives).
- Displays all conversations with status badges and orders by most recent.
- Processes all automated applications with --all-apps and generates PRDs for new applications or feature requests.
- Generates a feature request document from conversation messages and notifies admins; includes the application name.
- Handles conversations without an associated application.
- Can create a new internal application.
