## Purpose

Reqqy is a web-based Laravel application that uses conversational AI to gather feature requests and new application ideas, automatically generating professional PRDs from the conversation history. The primary users are end users (stakeholders) who request features or new apps and admins who review and approve/consolidate these requests. It solves the problem of capturing clear, comprehensive requirements from non-technical stakeholders by guiding discussions and producing ready-to-use documentation for development teams.

## Tech Stack

Laravel 12.0, PHP 8.4, livewire, flux, flux-pro

## Features (from tests)

- Generates a PRD document from conversation messages
- Generates a feature request document from conversation messages
- Creates a stub research document for the conversation
- Notifies all admin users when a document is created
- Dispatches a job when automated application is created
- Can create a new internal application
- Can create a new conversation when conversation_id is not provided
- Can send a message and receive llm response
- Passes conversation history to llm when generating response
- Dispatches generate new application prd job when signing off a new application conversation
- Handles conversations without an application
